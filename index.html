Systems Foundations

Machine-near work that still shapes how I think about runtime behavior.

Much of my earlier work lived close to the machine: ARM and MIPS board bring-up, boot order, firmware flows, U-Boot, and hardware-near debugging. That environment makes causality concrete.

It teaches you to think in state transitions, failure boundaries, resource constraints, and recovery paths. Software appears only after a chain of deterministic gates actually works.

That foundation still shows up in how I approach inference systems, memory movement, runtime orchestration, and control surfaces for large-model infrastructure.
